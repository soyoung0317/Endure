Q. ImGui에 대해 설명하시오
A.
	1. 정의
		간단하고 효율적인 GUI 라이브러리

	2. 2. 왜 ImGui를 사용하는가?
		- 빠른 프로토타이핑: ImGui는 코드 몇 줄로 빠르게 UI를 만들고 테스트할 수 있어, 프로토타이핑이나 디버깅 툴을 만들 때 매우 유용합니다.
		- 게임 개발: 게임 내 디버깅 도구, 에디터 툴, UI 등에서 널리 사용되며, 게임 화면에 영향을 주지 않으면서 빠르게 UI를 구성하고 조작할 수 있습니다.
		- 저사양 요구사항: ImGui는 매우 가벼운 라이브러리로, 고성능 게임이나 실시간 애플리케이션에서 사용하는 데 적합합니다.
		- 호환성: 다양한 그래픽 API(DirectX, OpenGL, Vulkan 등)와 호환되며, 다양한 플랫폼에서 사용할 수 있습니다.

	3. 장점
		- 간단하고 직관적: 코드 몇 줄만으로 UI를 만들고 수정할 수 있어 매우 직관적입니다.
		- 빠른 프로토타이핑: UI를 빠르게 만들고 수정할 수 있기 때문에 실험적인 툴이나 디버깅 툴을 만드는 데 유용합니다.
		- 디버깅 도구에 유용: 게임 개발에서 디버깅이나 에디터 툴, 프로파일링 툴을 빠르게 만들 때 적합합니다.
		- 플랫폼 호환성: 다양한 그래픽 API와 플랫폼을 지원하며, 크로스 플랫폼으로 개발할 수 있습니다.
	
	4. 단점
		- 전통적인 GUI와는 다름: 일반적인 GUI 프레임워크와 다르게 UI 상태를 명시적으로 저장하지 않기 때문에, 복잡한 UI를 만들기에는 다소 제한적일 수 있습니다.
		- 디자인 자유도 제한: UI 요소의 디자인 커스터마이징에서 전통적인 GUI보다 자유도가 낮을 수 있습니다. 더 정교한 디자인이나 복잡한 레이아웃을 구현하는 데 어려움이 있을 수 있습니다.
		- 메모리와 성능 이슈: 실시간 업데이트가 많을 경우, UI 요소가 많으면 성능이 저하될 수 있습니다.
