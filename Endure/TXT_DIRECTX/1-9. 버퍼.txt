Q. 인덱스 버퍼와 버텍스 버퍼에 대해 설명하시오.
A.
	1. 버텍스 버퍼 (Vertex Buffer)
		(1) 정의
			버텍스 버퍼는 정점(vertex) 데이터를 저장하는 메모리 공간입니다. 
			3D 그래픽스에서 정점은 객체의 위치, 색상, 법선 벡터, 텍스처 좌표 등과 같은 정보를 포함합니다.
			
		(2) 목적
			버텍스 버퍼는 3D 객체의 모양과 속성을 정의하는 중요한 정보를 담고 있으며, GPU에서 이 데이터를 사용하여 객체를 그립니다.
			
		(3) 내용
			버텍스 버퍼에는 객체의 모든 정점 정보가 포함됩니다. 예를 들어, 삼각형을 그린다면 그 삼각형을 이루는 세 개의 정점에 대한 위치와 색상 정보가 버텍스 버퍼에 저장됩니다.
			
		(4) 예시
			정점 위치: 각 정점의 x, y, z 좌표
			법선 벡터: 표면의 방향을 나타내는 벡터
			텍스처 좌표: 객체에 텍스처를 맵핑할 때 사용하는 좌표

	2. 인덱스 버퍼 (Index Buffer)
		(1) 정의
			- 인덱스 버퍼는 버텍스 버퍼의 데이터를 참조하는 인덱스 값을 저장하는 메모리 공간입니다. 이 버퍼는 복잡한 모델을 그릴 때 정점들을 재사용할 수 있도록 돕습니다.
			- 버텍스 버퍼를 통해 만들어진 정점 정보를 기반으로 각 정점의 순서를 정의하는 용도로 사용된다.

		(2) 목적
			인덱스 버퍼의 목적은 정점 데이터를 재사용하여 메모리와 계산 효율을 높이고, 하나의 정점 데이터를 여러 면(면체)에서 공유할 수 있게 하는 것입니다.
			
		(3) 내용
			인덱스 버퍼는 3D 객체의 구성 요소인 삼각형(혹은 다른 다각형)을 정의할 때, 버텍스 버퍼에 저장된 정점들의 순서를 지정합니다. 각 인덱스는 버텍스 버퍼에서 해당하는 정점을 가리키는 숫자입니다.

	3. 예시
		버텍스 버퍼
		: [ (x1, y1, z1, r1, g1, b1), (x2, y2, z2, r2, g2, b2), ... ]
		인덱스 버퍼 
		: [0, 1, 2, 1, 2, 3]  
		// 위 정점 버퍼를 참조하여 삼각형을 구성

Q. 깊이버퍼에 대해 설명하시오. 
A.
	1. 정의
		깊이 버퍼는 3D 그래픽스에서 물체의 화면상 깊이(카메라로부터의 거리)를 저장하는 버퍼이다. 
		보통 Z버퍼(Z-buffer)라고도 부르며, 렌더링할 때 픽셀마다 카메라에서 얼마나 떨어져 있는지(깊이값, Z값)를 기록해 두는 메모리 공간이다.

	2. 작동 원리
		렌더링 과정에서, 새로운 픽셀을 그리기 전에 해당 픽셀의 깊이를 깊이버퍼에서 확인합니다.
		만약 새로운 픽셀의 깊이 값이 현재 깊이버퍼에 저장된 값보다 작으면 (즉, 카메라에 더 가까우면), 해당 픽셀이 그려지고 깊이버퍼도 업데이트됩니다.
		반대로, 새로운 픽셀의 깊이 값이 더 크면 그 픽셀은 그려지지 않습니다.

	3. 장점
		- 정확한 렌더링 순서: 깊이버퍼를 사용하면 깊이 비교를 자동으로 수행하여, 겹치는 물체들을 올바르게 처리할 수 있습니다.
		- 복잡한 씬 렌더링 가능: 깊이버퍼를 통해 복잡한 씬에서도 물체의 앞뒤 관계를 자동으로 처리할 수 있습니다.

	4. 단점
		- 메모리 사용량: 깊이버퍼는 각 픽셀에 대해 깊이를 저장하기 때문에 화면 해상도가 높을수록 메모리 사용량이 증가합니다.
		- 정확도 문제: 깊이버퍼는 일반적으로 부동소수점으로 값을 저장하는데, 정밀도가 떨어지는 경우 깊이 값이 왜곡되어 z-fighting 현상이 발생할 수 있습니다. (예: 두 물체가 아주 가까워서 깊이 값이 동일하거나 거의 동일할 때, 경계가 깔끔하게 처리되지 않는 문제)
