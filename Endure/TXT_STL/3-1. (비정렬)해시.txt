Q. 체크섬(checksum)에 대해 설명하시오 

-------------------------------------------------

Q. 해시에 대해 설명하시오 
A.	
	1. 정의
		해시는 임의의 길이를 가진 데이터를 고정된 크기의 해시 값으로 변환하는 함수 혹은 기법을 말합니다.
		주로 해시 테이블(Hash Table)이라는 자료구조에서 키를 빠르게 검색하기 위해 사용됩니다.
	
	2. 특징
		- 평균적으로 검색, 삽입, 삭제가 O(1)의 시간 복잡도를 가집니다.
		- 해시 함수에 의해 키가 해시 값으로 변환되어 저장 위치를 결정합니다.
		- 서로 다른 키가 같은 해시 값을 가질 수 있는데, 이를 충돌(Collision)이라 하며, 충돌 해결이 필요합니다.
			충돌 해결 방식으로는 체이닝(Chaining), 개방 주소법(Open Addressing) 등이 있습니다.

	3. 사용하는 방법
		데이터를 저장할 때, 키를 해시 함수에 넣어 해시 값을 얻습니다.
		해시 값을 인덱스로 사용하는 배열에 데이터를 저장하거나 검색합니다.
		충돌이 발생하면, 체이닝 방식에서는 해당 인덱스에 연결 리스트를 만들어 추가 데이터를 저장하고,
		개방 주소법에서는 빈 공간을 찾아 데이터를 저장합니다.
		적절한 해시 함수 설계와 테이블 크기 조정으로 성능을 최적화합니다.

	4. 해시 관련 ox퀴즈
		좋은 해시는 키값과 비슷한값이 비슷한 해시값으로 나타나는것이다 (o , x)	
			-> (x) 좋은 해시는 키값이 조금 달라도 해시값은 최대한 균등하게 분포되도록 만들어야 합니다.
		해시는 역함수를 갖는다 (o, x)
			-> (x) 해시는 일반적으로 역함수(inverse function)를 갖지 않습니다.
				서로 다른 입력이 같은 해시 값을 가질 수 있어(충돌),해시 값만으로 원래의 키를 완벽하게 복원하는 역함수가 존재하지 않습니다.
		해시는 체크섬(checksum)과 유사한 개념이다 (o, x)
			-> (o) 두 기능 모두 임의 길이의 데이터를 고정된 크기의 값으로 변환해 데이터 무결성 확인이나 빠른 비교를 돕습니다.
				다만, 체크섬은 주로 데이터 전송 오류 검출에, 해시는 빠른 검색을 위해 사용되는 점이 다릅니다.
				+ 해쉬함수(해쉬알고리즘)이나 체크섬은 모두 파일의 변조를 예방하기 위한 수단으로 사용된다.

Q. 체크섬(checksum)에 대해 설명하시오 
A.
	1. 정의
		체크섬(CheckSum)은 데이터 전송이나 저장 과정에서 발생할 수 있는 오류를 검출하기 위해, 원본 데이터로부터 계산한 간단한 숫자 값입니다.
		
	2. 특징
		원본 데이터의 각 바이트 혹은 비트 값을 더하거나 특정 연산을 수행하여 고정된 크기의 값으로 만듭니다.
		데이터가 손상되거나 변조되면 체크섬 값이 달라져 오류를 감지할 수 있습니다.
		연산이 간단해 빠르게 계산 가능하지만, 단순 체크섬은 복잡한 오류를 완벽히 감지하지 못할 수 있습니다.
		
	3. 사용하는 방법
		송신 측에서 데이터 전체를 대상으로 체크섬 값을 계산하여 데이터와 함께 전송합니다.
		수신 측에서는 받은 데이터로 다시 체크섬을 계산해 송신 측 체크섬과 비교합니다.
		두 값이 다르면 데이터 오류가 발생한 것으로 판단하고 재전송을 요청하거나 오류 처리를 합니다.
