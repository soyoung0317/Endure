Q. 동기화 문제 (예: 서버와 클라이언트 간 상태 불일치) 해결 방법
A.
	1. 클라이언트가 서버와 동기화되지 않아 위치나 행동이 어긋남
		- 원인: 네트워크 지연, 패킷 손실, 클라이언트 예측 오류 등
		- 해결책:
			서버 권한 모델(Server Authority) 적용
			클라이언트 예측(Client Prediction)과 서버 보정(Server Reconciliation)
			주기적 상태 동기화 및 보간(Interpolation) 적용

	2. 지연(latency)과 패킷 손실 때문에 플레이어 행동이 지연되는 문제
		- 문제: 입력 딜레이가 길어져 조작감이 나빠짐
		- 해결책:
			클라이언트 예측 사용
			입력 큐 및 타임스탬프로 명령 처리
			서버 측에서 잘못된 행동 정정 시 클라이언트에 보정 신호 전송

	3. 서버 오버로드에 따른 동기화 지연 문제
		- 문제: 서버 처리 지연으로 게임 상태 업데이트가 늦어짐
		- 해결책:
			서버 부하 분산, 로드밸런싱
			중요 정보만 선별 동기화(델타 동기화)
			클라이언트 측에서 비중요 정보에 대한 로컬 예측 허용

	4. 동기화 보간(Interpolation)과 외삽(Extrapolation) 문제
		- 문제: 서버 업데이트가 불규칙하거나 느릴 때 클라이언트가 자연스럽게 움직임을 보여주기 어려움
		- 해결책:
			보간으로 위치 부드럽게 조정
			외삽으로 다음 위치 예측
			과도한 외삽 방지를 위한 보정
